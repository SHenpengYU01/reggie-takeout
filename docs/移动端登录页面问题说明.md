# 移动端登录页面说明

## 最新修复（2026-03-18）

✅ **登录跳转问题已修复！**

问题原因：`UserController` 使用 Redis 存储用户信息，但 `LoginCheckFilter` 检查的是 HttpSession，导致登录成功后被拦截器拦截并跳转回登录页面。

修复方案：修改 `UserController`，使用 HttpSession 而不是 Redis 来存储登录用户信息。

## 问题描述

当在桌面浏览器访问 http://8.136.220.203/front/index.html 时，会遇到以下问题：

1. 页面显示异常巨大，需要缩小才能看到内容
2. ~~无法正常进行登录操作~~ ✅ 已修复

## 原因分析

该页面是专门为移动端设计的：
- CSS中使用了 `rem` 单位配合响应式字体大小
- 在 `base.js` 中，当屏幕宽度 > 750px 时，根字体被设置为 28px
- 页面元素尺寸使用了很大的 `rem` 值（如 88rem 高度）
- 在桌面浏览器上，这会导致页面显示异常巨大

## 解决方案

1. 打开 Chrome/Edge 浏览器

4. 选择设备（如 iPhone 12 Pro）
5. 刷新页面，即可正常显示和使用
